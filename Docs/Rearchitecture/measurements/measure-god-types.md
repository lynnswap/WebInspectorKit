# measure:god-types

## Summary

Measured baseline for god-type analysis. Key correction to the task's naming assumption: there are NO types named DOMModel/NetworkModel/ConsoleModel/RuntimeModel/CSSModel — the files are named *Model.swift but each contains a single @MainActor @Observable class named *Session (DOMSession, NetworkSession, ConsoleSession, CSSSession) or RuntimeState; there is no session/model pair per domain. Each domain class is simultaneously (a) observable state store, (b) protocol command dispatcher (owns commandChannel + XxxProtocolCommands + recordError closure), and (c) protocol event sink (via XxxProtocolEventHandler conformances in extension files). The biggest god type is DOMSession: 26 stored properties (22 release + 4 DEBUG), a 1876-line main declaration spanning all of DOMModel.swift, plus 6 extensions across 6 other files including the 1768-line DOMSessionProtocolOperations.swift. The bind/unbindProtocolChannel triplet (commandChannel/protocolCommands/recordError) is copy-pasted across 5 domain classes plus CSSSession.StyleRefreshCoordinator — a clear owner-less invariant and re-measurement anchor for the rearchitecture.

## Findings

### 1. Naming clarification: *Model.swift files contain *Session/State types; no Model types exist

rg finds no 'class DOMModel' etc. Declarations: 'package final class DOMSession' (DOMModel.swift:8), 'package final class NetworkSession' (NetworkModel.swift:479), 'package final class ConsoleSession' (ConsoleModel.swift:307), 'package final class RuntimeState' (RuntimeModel.swift:412), 'package final class CSSSession' (CSSModel.swift:610). The 'Model' name survives only in file names and Sources/WebInspectorCore/Docs/*ModelResearch.md. Each class is both the domain state owner AND the protocol dispatcher: every one stores '@ObservationIgnored var commandChannel: ProtocolCommandChannel?', '@ObservationIgnored let protocolCommands: XxxProtocolCommands', '@ObservationIgnored var recordError: ((InspectorError?) -> Void)?'. Events flow in via XxxProtocolEventHandler conformances registered centrally in InspectorSession.configureProtocolEventDispatchers (AttachedInspection.swift:474-486).

Locations: Sources/WebInspectorCoreDOMCSS/DOM/DOMModel.swift:8; Sources/WebInspectorCoreConsoleNetwork/Network/NetworkModel.swift:479; Sources/WebInspectorCoreConsoleNetwork/Console/ConsoleModel.swift:307; Sources/WebInspectorCoreRuntime/Runtime/RuntimeModel.swift:412; Sources/WebInspectorCoreDOMCSS/CSS/CSSModel.swift:610; Sources/WebInspectorCore/AttachedInspection.swift:474

### 3. bindProtocolChannel/unbindProtocolChannel triplet copied across 6 owners — the strongest owner-less invariant signal

rg 'func bindProtocolChannel|func unbindProtocolChannel' Sources hits 6 distinct owners: DOMSessionProtocolOperations.swift:26/37, NetworkModel.swift:561/569, ConsoleModel.swift:334/342, RuntimeModel.swift:436/444, CSSModel.swift:649/653 (delegates to refreshCoordinator), CSSStyleRefreshCoordinator.swift:77/81. Each stores the same commandChannel + recordError pair; InspectorSession.bindProtocolChannel(for:) (AttachedInspection.swift:1020) and unbindProtocolChannel() (:1058) fan the channel out to all of them manually. A public Core surface would currently need to re-expose this per-domain wiring or move it behind a single channel owner.

Locations: Sources/WebInspectorCore/AttachedInspection.swift:1020; Sources/WebInspectorCoreDOMCSS/DOM/DOMSessionProtocolOperations.swift:26; Sources/WebInspectorCoreConsoleNetwork/Network/NetworkModel.swift:561; Sources/WebInspectorCoreConsoleNetwork/Console/ConsoleModel.swift:334; Sources/WebInspectorCoreRuntime/Runtime/RuntimeModel.swift:436; Sources/WebInspectorCoreDOMCSS/CSS/CSSStyleRefreshCoordinator.swift:77

### 4. AttachedInspection is a thin 5-let aggregate (42 lines); InspectorSession (712-line body, 8 stored props) is the actual lifecycle god type in that file

AttachedInspection (AttachedInspection.swift:322-363) stores only targetGraph/dom/network/runtime/console (all let) plus reset(); it is pure domain-state aggregation with no logic. InspectorSession (367-1078) stores attachment, lastError, and 6 @ObservationIgnored lifecycle fields (configuration, connectionPhase, protocolEventDispatchers, isDetaching, detachWaiters, attachRequestGeneration) and owns connect/detach generations, pump start/stop, bootstrap, dispatcher registry, and channel fan-out. Its domain accessors (dom/network/runtime/console, lines 393-407) are computed pass-throughs to attachment. Extensions: AttachedInspection.swift:9 (same file) and WebInspectorNativeTransport/InspectorSession+NativeAttachment.swift:5 (154-line WKWebView attach path).

Locations: Sources/WebInspectorCore/AttachedInspection.swift:322; Sources/WebInspectorCore/AttachedInspection.swift:367; Sources/WebInspectorNativeTransport/InspectorSession+NativeAttachment.swift:5
