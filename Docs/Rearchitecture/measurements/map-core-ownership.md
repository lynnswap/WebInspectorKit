# map:core-ownership

## Summary

Core ownership is cleanly layered but 100% package-scoped: WebInspectorCore and all its sub-targets plus WebInspectorTransport contain zero public declarations. InspectorSession (AttachedInspection.swift:367) owns connection lifecycle/pumps/bootstrap; AttachedInspection (AttachedInspection.swift:322) is a pure @Observable container of five long-lived domain sessions that are NEVER recreated across attach/detach cycles — teardown mutates them via reset(). Command issuing is already transport-decoupled: each domain session exposes perform(intent:) over an injected ProtocolCommandChannel, so an external consumer could observe state and issue commands if only AttachedInspection + the domain sessions + their vocabulary types became public; TransportSession/ProtocolCommandChannel can stay non-public. The one domain state living in the wrong layer for a Core public surface is pageUserInterfaceStyle, owned by WebInspectorUI's WebInspectorSession (UIKit WKWebView observer, not protocol-derived). Docs explicitly state the invariant that UI must not own transport/backend objects.

### 2. Q1 — AttachedInspection owns exactly five domain-session references, all `package let`, plus a reset() aggregate; it holds no lifecycle, transport, or connection state

Sources/WebInspectorCore/AttachedInspection.swift:320-363: `@MainActor @Observable package final class AttachedInspection { package let targetGraph: TargetGraph; package let dom: DOMSession; package let network: NetworkSession; package let runtime: RuntimeState; package let console: ConsoleSession; ... package func reset() { dom.reset(); network.reset(); runtime.reset(); console.reset() } }`. CSSSession is not a direct property — it is folded into DOMSession as `package private(set) var elementStyles: CSSSession` (DOMModel.swift:9), constructed in AttachedInspection.init line 337 via `DOMSession(targetGraph:elementStyles:)`. Note reset() does not reset targetGraph. Everything connection-related (transport, receiver, eventPump, reload closures, per-target enable state) lives in the private InspectorConnection (lines 237-262), not in AttachedInspection.

Locations: Sources/WebInspectorCore/AttachedInspection.swift:322; Sources/WebInspectorCore/AttachedInspection.swift:357; Sources/WebInspectorCoreDOMCSS/DOM/DOMModel.swift:9

### 3. Q2 — Domain sessions are created once and mutated (reset + rebind) across attach/detach cycles; AttachedInspection is never recreated. Per-cycle identity lives in InspectorConnection

InspectorSession.init resolves `attachment ?? AttachedInspection()` into a `package let attachment` (AttachedInspection.swift:368, 409-422) — identity is stable for the life of InspectorSession. Per attach cycle: connect() creates a new private InspectorConnection (line 549), sets connectionPhase = .pending, and injects a fresh ProtocolCommandChannel into each domain session via bindProtocolChannel (lines 1020-1056: dom/network/runtime/console.bindProtocolChannel(channel, recordError:)). Teardown (performAttachmentTeardown, lines 663-682) unbinds the channel, closes receiver, stops pumps, detaches transport, then calls resetAttachmentModels() (lines 705-710) which calls reset() on dom/network/runtime/console — same object instances, state wiped. The connect failure path (lines 583-601) does the same reset inline. Consequence for the design doc: external consumers can hold long-lived references to domain sessions across reattaches (Observation-friendly), but every attach cycle erases accumulated domain state; command availability is signaled via dom.recordCommandAvailabilityMutation() and channel currency checks, and stale commands fail with TransportSession.Error.transportClosed (ProtocolCommandChannel.swift:48-50).

### 4. Q3 — Minimal publicization set: AttachedInspection + 5 domain sessions + intent/model vocabulary + a public accessor on WebInspectorSession; transport can stay non-public because domain sessions already encapsulate command sending

Today a custom tab receives only WebInspectorSession (WebInspectorTab.swift:60-70 factory signature `(WebInspectorSession) -> UIViewController`), whose public members are just pageUserInterfaceStyle/attach/detach; `inspector: InspectorSession` and `attachment: AttachedInspection` are package (WebInspectorSession.swift:12, 53). Core targets have literally zero `public` declarations (rg across WebInspectorCore*, WebInspectorTransport, WebInspectorNativeTransport: only os.log `.public` privacy hits). ISSUE-commands path is already transport-free at the consumer boundary: each session exposes `package func perform(_ intent:)` — RuntimeCommand.Intent (RuntimeModel.swift:450), ConsoleCommand.Intent (ConsoleModel.swift:348), NetworkCommand.Intent (NetworkModel.swift:575), DOMCommand.Intent (DOMSessionProtocolOperations.swift:164), CSSCommand.Intent (CSSModel.swift:658) — all routed through the injected ProtocolCommandChannel, which enforces currency/attachment. OBSERVE path: sessions are @MainActor @Observable with revision counters (DOMSession.treeRevision/selectionRevision/commandAvailabilityRevision DOMModel.swift:11-14; NetworkSession.requestTopology/Content/DisplayRevision NetworkModel.swift:483-485; ConsoleSession.warningCount/errorCount ConsoleModel.swift:308-309; RuntimeState.selectedContextKey RuntimeModel.swift:413). Therefore the minimal public surface is: (a) a public accessor from WebInspectorSession to AttachedInspection, (b) AttachedInspection and the five session classes' read/observe/perform members, (c) the Sendable vocabulary types they mention (DOMNode, NetworkRequest, ConsoleMessage, RuntimeContext/RuntimeRemoteObject, ProtocolTarget.ID, ProtocolCommand.Result, *Command.Intent), (d) optionally InspectorSession.hasActiveConnection/lastError/canReloadPage/reloadPage for lifecycle-adjacent UI. NOT needed public: TransportSession, TransportBackend, TransportReceiver, ProtocolCommandChannel, DomainEventPump, InspectorConnection. Measured surface cost: package member counts are DOMModel 72, NetworkModel 105, ConsoleModel 62, RuntimeModel 61, CSSModel 80, TargetGraph 34 — so an unfiltered publicization is large; a curated public facade or @_spi-style tiering is the realistic option. Structural blocker: the vocabulary lives in three sub-targets re-exported into WebInspectorCore via `@_exported import` (AttachedInspection.swift:3-6) with `package` visibility — public re-export requires making those sub-target declarations public (or collapsing targets), and command results also surface WebInspectorTransport types (ProtocolCommand.Result, ProtocolTarget), currently package in the Transport target.

### 6. Supporting detail — attach flow and reload closures are injected from WebInspectorNativeTransport, keeping WKWebView knowledge out of Core

InspectorSession+NativeAttachment.swift:5-58: `package extension InspectorSession { func attach(to webView: WKWebView) }` resolves native symbols, builds TransportReceiver + NativeInspectablePage, then calls the package staging API on InspectorSession (beginAttachmentRequest / detachForAttachmentRequest / makeTransportSession / connectAttachment) passing `pageReloadAction: { try page.reload() }`, `pageReloadAvailability: { page.canReload }`, `connectionCleanup: { page.restoreInspectabilityIfNeeded() }`. Core's reload surface (canReloadPage AttachedInspection.swift:712, reloadPage :720) is therefore WKWebView-agnostic — a good precedent for the public design: the same closure-injection seam can serve a second app or an AppKit host. reloadPage also performs the full model reset + reseed sequence (lines 729-736: prepareForPageReload, reset x4, seedDOMSession/seedRuntimeState from transport snapshot, syncTargets).
