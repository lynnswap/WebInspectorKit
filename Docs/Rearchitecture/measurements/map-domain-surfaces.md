# map:domain-surfaces

## Summary

4ドメイン(DOM/CSS・Network・Console・Runtime)の session 型はすべて @MainActor @Observable の package final class で、public 宣言は Core 系 6 target に 0 件(実測)。読み取り面は「@Observable な live model ツリー + revision ベースの差分 API + Sendable snapshot()」の三層構造で統一されている。コマンド面は各 session の `perform(_ intent:) async throws -> ProtocolCommand.Result` を共通の底とし、その上に少数の高レベル async メソッド(fetchResponseBody / evaluate 経由 / highlightNode / deleteNode 等)が乗る。ただし現状の package 面は (1) NetworkRequest.request/response/metrics など raw protocol Payload 構造体を observable state としてそのまま保持・公開、(2) perform の戻り値が resultData: Data を持つ生エンベロープ ProtocolCommand.Result、(3) apply*(プロトコルイベント注入用ミューテータ、Network 18 / DOM 19 / Runtime 8 / Console 5 個)が読み取り面と同じ型に同居、という 3 点で protocol 層が全面的に漏れており、public 化には「payload の semantic 型化」「Result を返さない command wrapper」「apply* の隔離」が必要。Console は clearMessages 等の Intent は揃っているが高レベル wrapper が無く、UI target からの利用箇所は 0 件(カスタム Console タブが最初の消費者になる)。DOM の変異 intent は removeNode/undo/redo のみで、属性編集・outerHTML 編集の intent は存在しない。
